# AI小说剧情吸引力工程化方案

正文写得好，只能解决“这一章读起来顺不顺”。剧情足够吸引人，解决的是“读者为什么必须看下一章”。两者不是一回事。剧情吸引力要分三级处理：`05 故事主轴` 决定全书是否值得追，`05a/05b` 决定幕级追读曲线是否成立，`12 剧情梗概设计` 决定单章是否抓人。

## 一、吸引力的本质

剧情吸引人，不是事件多，也不是反转多，而是持续制造读者的心理欠账：

1. 好奇欠账：读者知道有东西没揭开。
2. 利益欠账：角色想要的东西还没拿到，或者拿到会付代价。
3. 情绪欠账：关系裂缝、羞辱、误解、亏欠没有结清。
4. 爽点欠账：主角有机会反击，但还差最后一步。
5. 危机欠账：角色解决了眼前问题，却打开更大的风险。

一个章节剧情如果没有欠账，就算文笔好，也容易像“完成任务”。

## 二、三级剧情吸引力门禁

### 1. 全书主轴门禁

位置：`05 故事主轴 -> Q7 故事主轴吸引力评审 -> 05a 幕次框架`

它判断整本书有没有长期追读理由：

- 核心问题是否能贯穿全书。
- 主角长线欲望是否强烈且会变化。
- 终局代价是否具体。
- 阻力系统是否可持续。
- 中后段是否有足够强的认知反转。

### 2. 幕次结构门禁

位置：`05a 幕次框架 -> Q8 幕次框架吸引力评审 -> 05b 幕次核心骨架`

它判断全书追读曲线是否逐幕升高：

- 每一幕是否有不同结构功能。
- 幕与幕之间是否持续加压。
- 大中小钩子是否分布合理。
- 每幕是否既有兑现也有新欠账。
- 中段是否不会塌。

### 3. 幕级节点门禁

位置：`05b 幕次核心骨架 -> Q9 幕次核心骨架吸引力评审 -> 开篇/循环`

它判断一整幕的节点链是否能支撑连续章节：

- Hook/Setup/Midpoint/Crisis/Climax/Resolution 是否逐级升级。
- 中点是否改变认知、目标、主动权或风险等级。
- 危机是否有真实代价。
- 高潮是否兑现本幕承诺。
- 幕末是否留下下一幕追读理由。

### 4. 单章剧情门禁

位置：`12 剧情梗概设计 -> Q4 剧情吸引力评审 -> Q5 剧情强化 -> Q6 钩子账本 -> 13 写作指南`

它判断本章是否值得写。

## 三、单章剧情吸引力评分

每章剧情梗概用 100 分制，门禁位置是 `12 -> Q4 -> Q5(可选) -> Q6 -> 13`。

| 维度 | 权重 | 通过标准 | 典型失败 |
| --- | ---: | --- | --- |
| 核心钩子 | 20 | 一句话能说清“为什么非看不可” | 只是发生了一些事 |
| 欲望与代价 | 15 | 主角想要明确，得到/失去都有代价 | 目标弱，选择无损失 |
| 阻力升级 | 15 | 阻力比上一章更难或更复杂 | 同类障碍重复出现 |
| 反转与发现 | 15 | 至少有一次认知变化、局势翻转或信息差打开 | 线性推进，无惊讶 |
| 因果链 | 10 | 事件由角色选择触发，不靠巧合搬运 | 作者安排感强 |
| 爽点/痛点回报 | 10 | 本章给读者一个明确回报 | 全是铺垫，没有兑现 |
| 章末追读 | 10 | 结尾打开更强问题 | 结尾像总结 |
| 新鲜度 | 5 | 有本书独有的设定用法或人物处理 | 套路可替换 |

门禁建议：

- `>= 85`：剧情可进入写作指南。
- `75-84`：补强章末钩子、代价或信息差。
- `60-74`：重构本章事件链。
- `< 60`：回退 `11 剧情生成方向指导`，因为方向本身不够吸引人。

## 四、剧情工程模块

### 1. 钩子账本

每个项目维护一个 `05_quality/剧情钩子账本.md`，记录所有未结清的读者欠账。

账本不等于大纲，它只回答：

- 当前有哪些未揭示悬念？
- 哪些爽点已经承诺但还没兑现？
- 哪些关系裂缝还没处理？
- 哪些伏笔快过期，再不推进会被读者遗忘？
- 下一章最应该打开哪一个问题？

### 2. 情绪股票

每条剧情线都像股票，有买入、有拉升、有兑现。AI 常见问题是只买不卖，或者刚买就卖。

规则：

- 小钩子 1-2 章内要有反馈。
- 中钩子 3-8 章内必须推进一次。
- 大悬念可以长期保留，但每幕至少要给一次新解释或新误导。
- 爽点承诺不能无限拖，拖太久会变成水。

### 3. 冲突升级阶梯

同一类冲突不能连续重复。升级方向可以是：

```text
信息更少 -> 对手更强 -> 代价更高 -> 关系更痛 -> 规则更残酷 -> 选择更不可逆
```

每章至少占一个升级方向。

### 4. 章节剧情四件套

`12 剧情梗概设计` 的合格产物必须隐含四件事：

```yaml
chapter_attraction:
  promise: 本章向读者承诺什么回报
  pressure: 什么压力逼角色行动
  turn: 中途发生什么认知/局势变化
  debt: 章末留下什么未结清欠账
```

缺少 `turn`，剧情会平。缺少 `debt`，读者不会追。

## 五、工具步骤

新增六个剧情吸引力工具：

- `Q4`：剧情吸引力评审，评估 `剧情v{round}.md` 是否值得进入写作。
- `Q5`：剧情强化重构，重写剧情梗概，不碰正文。
- `Q6`：剧情钩子账本沉淀，把本章打开/推进/兑现的钩子写入项目账本。
- `Q7`：故事主轴吸引力评审，评估全书级追读理由。
- `Q8`：幕次框架吸引力评审，评估幕与幕之间的追读曲线。
- `Q9`：幕次核心骨架吸引力评审，评估单幕节点链。

推荐手动验证：

```bash
python run_workflow.py single Q4 -p <小说名> -g <题材>
python run_workflow.py single Q5 -p <小说名> -g <题材>
python run_workflow.py single Q6 -p <小说名> -g <题材>
python run_workflow.py single Q7 -p <小说名> -g <题材>
python run_workflow.py single Q8 -p <小说名> -g <题材>
python run_workflow.py single Q9 -p <小说名> -g <题材>
```

推荐自动接入：

```text
主轴：05 -> Q7 -> 05a
幕框架：05a -> Q8 -> 05b
幕骨架：05b(每幕) -> Q9(每幕)
开篇：06 -> 07 -> Q4 -> Q5(可选) -> Q6 -> 08 -> 09
循环：11 -> 12 -> Q4 -> Q5(可选) -> Q6 -> 13 -> 14
```

## 六、最优解

正文质量闭环解决“怎么写好”。剧情吸引力闭环解决“写什么才值得写”。

完整生产线应变成：

```text
故事主轴 -> 主轴吸引力评审
-> 幕次框架 -> 幕次吸引力评审
-> 幕次核心骨架 -> 幕骨架吸引力评审
-> 剧情方向 -> 剧情梗概 -> 单章吸引力评审 -> 剧情强化 -> 钩子账本
-> 写作指南 -> 正文生成 -> 正文质量评审 -> 定向重写 -> 状态文档
```

对现有项目来说，真正的完整生产线是：

```text
剧情方向 -> 剧情梗概 -> 吸引力评审 -> 剧情强化 -> 钩子账本
-> 写作指南 -> 正文生成 -> 正文质量评审 -> 定向重写 -> 状态文档
```

这样故事会先过“全书追读理由”门，再过“幕级追读曲线”门，再过“单章追读价值”门，最后才进入“文字呈现”门。长期来看，项目会形成两个记忆系统：`剧情钩子账本` 管读者欠账，`风格记忆` 管表达退化。
