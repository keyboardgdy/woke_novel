# AI小说好看工程化方案

目标不是让提示词更华丽，而是把“好看”变成一个可诊断、可重写、可沉淀的生产系统。现有流程已经有创意、世界观、人物、主轴、剧情、指南、正文、状态文档。完整缺口有两个：剧情梗概进入写作前缺少“追读价值”门禁，正文生成后缺少“表达质量”门禁。

## 一、核心判断

AI 小说不好看，通常不是“文笔不够”，而是四类系统性问题叠加：

1. 欲望弱：角色没有强烈且持续的想要，剧情像任务清单。
2. 阻力弱：行动代价低，选择没有损失，读者不用担心。
3. 变化弱：场景结束后人物关系、信息差、局势没有明显改变。
4. 语言假：句子在解释情绪、复述设定、制造高级感，而不是让人物活在现场。

因此最优解不是继续堆“写得更精彩”，而是增加两道门禁：

```text
05 故事主轴 -> 主轴吸引力评审 -> 05a 幕次框架
05a 幕次框架 -> 幕次吸引力评审 -> 05b 幕次核心骨架
05b 幕次核心骨架 -> 幕骨架吸引力评审 -> 开篇/循环
12 剧情梗概设计 -> 剧情吸引力评审 -> 剧情强化 -> 13 写作指南
14 正文创作 -> 正文质量评审 -> 定向重写 -> 15 状态文档
```

其中正文质量闭环让系统每章都做：

```text
正文生成 -> 质量评审 -> 问题定位 -> 定向重写 -> 复评 -> 状态沉淀
```

## 二、质量指标体系

每章用 100 分制，但分数只用于门禁，真正重要的是缺陷定位。

| 维度 | 权重 | 通过标准 | 典型失败 |
| --- | ---: | --- | --- |
| 角色驱动 | 20 | 主角和关键角色的行动都能追溯到欲望、恐惧、利益 | 角色只是完成剧情节点 |
| 戏剧冲突 | 20 | 每个核心选择都有代价，阻力来自人、规则、环境或自我矛盾 | 一路顺滑、靠巧合推进 |
| 场景变化 | 15 | 每场结束后至少一个状态改变：信息、关系、资源、位置、风险 | 场景只表达情绪 |
| 读者钩子 | 15 | 章末留下未完成动作、未揭示信息或情绪缺口 | 结尾像阶段总结 |
| 人物声音 | 10 | 主要角色的台词和动作可区分 | 所有人都像同一个作者 |
| 叙事密度 | 10 | 每段要么推进剧情，要么塑造人物，要么埋/收伏笔 | 水段、重复情绪 |
| 语言质感 | 10 | 少解释，多动作、细节、节奏变化 | AI 腔、金句癖、形容词堆砌 |

门禁建议：

- `>= 85`：直接进入状态文档。
- `75-84`：轻量润色，只改语言和节奏，不动事件。
- `60-74`：局部重写，至少重写 1-2 个低分场景。
- `< 60`：回退到 `12 剧情梗概设计` 或 `13 写作指南`，因为正文问题已经来自上游结构。

## 三、系统架构

### 1. 新增目录

建议每个项目增加：

```text
projects/<小说名>/
├── 05_quality/
│   ├── 质量评审vN.md
│   ├── 重写指令vN.md
│   ├── 复评vN.md
│   └── 风格记忆.md
```

`05_quality` 不参与正文阅读导出，只服务生成质量闭环。

### 2. 新增工具步骤

已补充三个工具模板，并注册为可执行步骤：

- `Q1`：`steps/tools/Q1 章节质量评审.md`
- `Q2`：`steps/tools/Q2 章节定向重写.md`
- `Q3`：`steps/tools/Q3 风格记忆沉淀.md`

推荐接入点：

```text
开篇：06 -> 07 -> 08 -> 09 -> 质量评审 -> 定向重写(可选) -> 复评 -> 10
循环：11 -> 12 -> 13 -> 14 -> 质量评审 -> 定向重写(可选) -> 复评 -> 15 -> 16
```

先以手动 `single/session` 调用验证模板质量，再改 `run_workflow.py` 自动编排。

### 3. 状态反哺

把每章低分原因写入 `05_quality/风格记忆.md`，下一章 `13 写作指南` 读取它，形成项目级写作指纹。

不建议把所有评审意见直接塞进正文 prompt。正确方式是沉淀为短规则：

```yaml
style_memory:
  strengths:
    - "角色在沉默中转移注意力时最有真实感，保留这种处理。"
  recurring_failures:
    - "容易在章末解释人物关系变化，下一章必须改成未完成动作。"
  banned_patterns:
    - "不要用'他终于明白'总结成长。"
  next_chapter_bias:
    - "下一章优先制造信息差，而不是继续写情绪余波。"
```

## 四、生成策略

### 1. 从“章”改成“场景资产”

正文前，`13 写作指南` 应要求模型列出每个场景的四元组，但不写进正文：

```yaml
scene_contract:
  goal: 角色此刻要什么
  obstacle: 谁或什么阻止他
  turn: 场景中途发生什么不可逆变化
  exit_hook: 场景以什么缺口离开
```

如果一个场景没有 `turn`，它就不该存在，或应并入其他场景。

### 2. 先强情节，后强文笔

工程优先级：

```text
因果链 > 人物选择 > 冲突代价 > 场景变化 > 语言质感
```

文笔不能救结构。评分低于 70 时不要做润色，应该重写场景目标、阻力和转折。

### 3. 每章一个主爽点或主痛点

网文好看需要稳定回报。每章至少有一个明确读者回报：

- 爽点：主角主动破局、反制、升级、识破、夺回主动权。
- 痛点：误解加深、重要关系受损、代价揭示、旧伤被触发。
- 悬疑点：新线索出现，但解释权被推迟。
- 情绪点：角色做出与以往不同的选择。

禁止一章同时追求太多回报，最后会变成平均用力。

### 4. 用“缺陷码”驱动重写

评审输出不要只写“节奏差”，要给缺陷码：

| 缺陷码 | 含义 | 重写动作 |
| --- | --- | --- |
| D01 | 角色无目标 | 给角色添加明确短期目标 |
| D02 | 阻力无效 | 增加具体对手、规则或代价 |
| D03 | 场景无变化 | 改写结尾，让信息/关系/局势改变 |
| D04 | 对话传声筒 | 改成试探、防御、隐瞒或误导 |
| D05 | 情绪解释 | 删除结论句，改为动作和身体反应 |
| D06 | 伏笔遗忘 | 回收或推进一个已存在伏笔 |
| D07 | 章末无钩子 | 留未完成动作、未说完的话或新风险 |
| D08 | 人物降智 | 补出角色合理误判的信息边界 |
| D09 | AI腔 | 删除总结、金句、抽象形容词 |
| D10 | 节奏单一 | 插入快慢切换或场景压缩 |

## 五、自动化落地路线

### 阶段 1：无代码验证

1. 生成一章正文。
2. 手动运行 `python run_workflow.py single Q1 -p <小说名> -g <题材>` 评审。
3. 如果低于 85，运行 `python run_workflow.py single Q2 -p <小说名> -g <题材>` 改正文。
4. 运行 `python run_workflow.py single Q3 -p <小说名> -g <题材>` 更新风格记忆。

目标：验证评分标准是否能稳定指出“为什么不好看”。

### 阶段 2：半自动命令

新增命令：

```bash
python run_workflow.py polish -p <小说名> --round 12
```

内部执行：

```text
quality_review(round)
if score < threshold:
    rewrite(round)
    quality_review(round, phase=after_rewrite)
style_memory_update(round)
```

### 阶段 3：全流程门禁

在 `run_workflow_from_cursor` 中，把质量门插在 `09/14` 之后、`10/15` 之前。这样状态文档读取的是最终正文，而不是初稿。

### 阶段 4：多候选正文

对关键章节可生成 2-3 个正文候选：

```text
同一剧情 + 同一指南 -> 多正文候选 -> 质量评审排序 -> 选择最高分 -> 定向重写
```

这比单稿反复润色更有效，因为模型第一次生成时容易陷入局部最优。

## 六、推荐默认阈值

```yaml
quality_gate:
  pass_score: 85
  light_rewrite_score: 75
  structural_rewrite_score: 60
  max_rewrite_rounds: 2
  never_rewrite:
    - "只因为文风偏好不同"
    - "评审没有引用正文证据"
  must_rewrite:
    - "章末无钩子"
    - "核心角色行动无动机"
    - "场景结束没有状态变化"
```

## 七、最小代码改造点

需要改动的位置：

1. `project_structure.py`：新增 `05_quality` 目录。
2. `path_resolver.py`：新增 `{quality}` 变量，映射到项目的 `05_quality`。
3. `workflow_runner.py`：增加工具步骤映射，例如 `Q1/Q2/Q3`、`Q4/Q5/Q6` 与 `Q7/Q8/Q9`。
4. `run_workflow.py`：在 `05/05a/05b/12` 后插入剧情吸引力门禁，在正文步骤后插入质量门禁。
5. `i18n/zh.json`、`i18n/en.json`：补步骤名。

推荐先不要改变主流程，先用工具模板跑 3-5 章样本，确认评审不胡评、重写不跑偏后再自动化。

## 八、真正的最优解

最优解不是“一个超级提示词”，而是：

```text
结构化上游设计
+ 主轴/幕次/幕骨架吸引力门禁
+ 剧情吸引力门禁
+ 剧情钩子账本
+ 正文生成
+ 严格质量门禁
+ 定向重写
+ 项目级风格记忆
+ 下一章反哺
```

这套系统的关键是闭环。只要评审能稳定识别缺陷，重写能只改缺陷不乱改剧情，风格记忆能持续压制复发问题，AI 小说会从“偶尔有好段落”变成“整体稳定可读，关键章节能好看”。
