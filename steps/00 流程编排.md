会话1

执行01 创意方案生成（循环 option_count 次，生成 option_count 个创意方案）；用户选择一个创意方案，执行步骤02 创意方案补充（追加至创意方案_{n}.md）；
选完后抽取小说名并把项目目录改名为最终小说名，抽取参考作品并写入项目档案。

会话2

执行03 世界观与设定生成；执行04 人物档案与人物关系；

会话3

执行05 故事主轴；执行05a 幕次框架；执行05b 幕次核心骨架（每幕执行一次）；
05b 全部完成后执行步骤18（phase=post_05b）生成项目根 CLAUDE.md。

会话4

执行06 开篇剧情提取（输出"剧情方向v1.md"）；执行07 开篇剧情梗概（输出"剧情v1.md"）；执行08 开篇写作指南（输出"写作指南v1.md"）；执行09 开篇正文创作（输出"正文v1.md"）；执行10 状态文档（输出"状态v1.md"）；
开篇完成后用 `extract_and_create_story_summary` 把首章梗概写入 `{state}/故事总梗概.md` 初始版本。

创作循环（轮次从2开始）

按幕次和章节数迭代：每幕有指定章节数，第一幕开篇已完成所以该幕循环 act_chapters-1 次，其余幕循环 act_chapters 次。
每轮执行：执行11 剧情生成方向指导（输出"剧情方向v{round}.md"）；执行12 剧情梗概设计（输出"剧情v{round}.md"）；执行13 写作指南（输出"写作指南v{round}.md"）；执行14 正文创作（输出"正文v{round}.md"）；执行15 状态文档（输出"状态v{round}.md"）；执行16 故事梗概精简（在 round session 内紧跟 15 跑，输出至 `{state}/故事总梗概.md`，并由 `sync_summary_to_state` 把精简版写回本轮末轮的状态v{round}.md）。

一幕完成后执行：执行17 幕次故事梗概精简（输出至 `{state}/故事总梗概.md`，并 `sync_summary_to_state` 写回末轮状态文档）；随后执行步骤18（phase=post_17）刷新项目根 CLAUDE.md。

## 目录结构

```
{project}/
├── 00_baseline/     # 创意方案、根底设定
├── 01_plots/         # 剧情相关（含开篇的"剧情方向v1.md"）
├── 02_guides/        # 写作指南
├── 02_output/        # 正文输出
├── 03_state/         # 状态文档、故事总梗概
├── 04_characters/    # 人物档案
└── CLAUDE.md         # 步骤18 生成的项目级说明（文件，不是目录）
```

> 日志位于仓库根 `logs/<小说名>/`，与 `projects/` 平级，不在项目目录里。
