## 核心任务

基于创意方案和世界观，生成核心人物档案与关系矩阵，输出为结构化 JSON。

**设计原则**：人物不是数据表，是"一组矛盾的集合体"。好角色的标志是——他最想要的东西和他最害怕的东西指向同一个方向。

---

## 设计指导

### 什么让角色值得写

- 欲望和恐惧之间有张力：他追求的东西恰好需要面对他最回避的东西
- 自我认知有盲区：他以为自己是什么样的人，和他实际行为之间有缝隙
- 有独立的利益账本：他不是为主角服务的工具，他有自己要争取的东西
- 行动逻辑自洽：给定他的过往和性格，他的每个选择读者事后会觉得"确实只能这样"

### 角色层级

- **核心角色**（3–5人）：完整填写所有字段，每个字段认真设计
- **重要配角**（2–4人）：填写基础字段 + 与主线相关的关系字段，可省略部分亲密细节字段
- **不要**为了凑数生成无后续功能的边缘角色

### 字段填写标准

- 普通字段 1–2 句，复杂字段最多 3 句
- 数组字段只保留最有复用价值的 2–5 项
- 所有描述必须具体——"害怕被抛弃"不如"害怕在自己终于开口求助时对方已经转身走了"

---

## 边界约束

人物档案只描述「角色是怎样的人」和「关系天然会如何发热、退缩、欠债」：

- ❌ 不设计章节级具体行动与决策
- ❌ 不设计完整冲突事件链
- ❌ 不设计已完成的成长弧线
- ❌ 不设计特定章节的成文场景
- ✅ 描述角色的本质矛盾、行为倾向、关系起点和可能的变化方向

## 人物档案 JSON 结构

每个核心人物包含以下字段：

```json
{
  "name": "",
  "role": "",
  "surface_traits": [],
  "background": "",
  "ability_or_resource": "",
  "core_desire": "",
  "core_fear": "",
  "emotional_wound": "",
  "unspoken_need": "",
  "shame_point": "",
  "moral_boundary": "",
  "relationship_habit": "",
  "intimacy_trigger": "",
  "intimacy_defense": "",
  "sensual_appeal": "",
  "physical_presence": "",
  "attractive_details": [],
  "self_awareness_of_appeal": "",
  "sexual_boundary": "",
  "consent_language": "",
  "after_intimacy_response": "",
  "body_language_when_moved": "",
  "body_language_when_defensive": ""
}
```

**关键字段说明**（只解释易误解的）：

| 字段 | 要写什么 | 不要写什么 |
|------|----------|-----------|
| emotional_wound | 最容易被刺痛的旧缺口 | 悲惨履历堆砌 |
| unspoken_need | 最想从别人那得到但羞于承认的 | 空泛的"被爱" |
| shame_point | 被看见就会防御/攻击/逃走的弱点 | 普通缺点 |
| intimacy_trigger | 让角色动摇的具体情境 | "被温柔对待"之类空泛描述 |
| sensual_appeal | 吸引力的来源质感（力量/松弛/危险/克制/反差…） | "很帅/很美" |
| physical_presence | 进入空间时的身体存在感（姿态/步伐/声音/气味） | 静态外貌清单 |
| attractive_details | 2–5个会被他人反复注意的可感细节 | "五官精致""身材好" |
| consent_language | 表达愿意/犹豫/拒绝时符合其性格的方式 | 通用模板 |

## 关系矩阵 JSON 结构

每对重要关系包含以下字段：

```json
{
  "pair": ["角色A", "角色B"],
  "surface_relation": "",
  "inner_relation": "",
  "attraction_source": "",
  "sexual_attraction_source": "",
  "gaze_pattern": "",
  "desire_tells": [],
  "repulsion_source": "",
  "power_balance": "",
  "relationship_debt": "",
  "unspoken_rule": "",
  "intimacy_boundary": "",
  "sexual_threshold": "",
  "consent_risk": "",
  "boundary_crossing_risk": "",
  "slow_burn_path": [],
  "abstract_intimacy_motifs": [],
  "possible_sexual_payoff": "",
  "delayed_payoff": ""
}
```

**关键字段说明**：

| 字段 | 要写什么 | 不要写什么 |
|------|----------|-----------|
| inner_relation | 真实的牵引、误解、依赖、戒备或欲望 | 重复 surface_relation |
| sexual_attraction_source | 具体的吸引力来源（身体反差/气味/声音/被需要…） | "互相喜欢" |
| gaze_pattern | 被吸引时视线落在哪里、如何移开掩饰 | 器官清单式凝视 |
| desire_tells | 欲望外泄的小动作（话变少/动作放慢/突然刻薄…） | 直接说"心动了" |
| relationship_debt | 尚未还清的情感欠账 | 泛泛的"感情深" |
| slow_burn_path | 升温阶段（只写阶段名和代价） | 具体章节剧情 |
| abstract_intimacy_motifs | 适合反复出现的亲密动作/物件/距离 | 直接的性描写 |
| consent_risk | 同意表达最容易含混的风险点 | 无视此字段 |

## 底线原则

所有旖旎、性感吸引和性亲密设计必须建立在：成年角色、清醒自愿、关系逻辑成立的前提上。

设计重点是人物视角、欲望、同意、羞耻、选择和关系后果——不是把角色简化为器官清单或占有对象。允许描写男性和女性的性感吸引力，允许故事中真实发生性亲密，但人物的边界、尊严和行动逻辑不可为此让步。
