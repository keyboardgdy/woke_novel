基于故事主轴，进行专业的幕次规划——把宏观骨架切成可执行的结构单元。

---

## 设计原则

幕次框架的任务是**节奏建筑**。每一幕是一个完整的压力循环：积蓄 → 升级 → 爆发 → 余波。幕与幕之间不是平铺，而是阶梯——每一幕结束时的基准线比上一幕高。

好的幕次划分标准：
- 每一幕有自己独立的核心冲突（不是全书冲突的简单重复）
- 幕末必须发生"不可逆变化"——回不去上一幕的状态
- 相邻两幕的情绪色温有明显差异（不是匀速推进）
- 删掉任何一幕，后续幕次的逻辑链会断裂

---

## 输出约束

- 每一幕只写结构功能、核心冲突、章节预算、人物/关系变化和幕末钩子
- 不展开到单章剧情，不写正文场景，不重复故事主轴全文
- 不要为了"看起来专业"堆砌结构术语——每一条都要有具体内容

---

## 一、宏观结构模型

根据故事特性选择最适合的结构模型（三幕式/英雄之旅/多幕螺旋/双线交织/其他），说明选择理由：
- 这个故事为什么适合这种结构？
- 这种结构如何服务核心冲突的升级节奏？

## 二、篇幅与章节预算

目标全书规模：{novel_size}，目标总字数：{target_word_count}。

章节分配规则：
- 单章正文目标 2500–4500 字，默认约 3000 字
- 先用目标总字数反推全书章节总量，再分配到各幕
- 每幕章节数由内容复杂度、冲突层级、角色弧深度决定
- 所有幕合计章节数必须能支撑目标总字数
- 高潮幕/关系转折幕可加章节，过渡回落幕应压缩
- 每一幕标注：预计章节数、预计字数区间、篇幅理由

## 三、幕次详细规划

每一幕包含：

| 要素 | 内容 |
|------|------|
| 幕次编号与名称 | 动词短语命名（如"撕裂信任"而非"第二幕"） |
| 结构功能 | 这一幕在全书节奏中的位置（铺设/升级/转折/高潮/收束） |
| 核心冲突 | 本幕的独立矛盾——与全书核心冲突的关系是什么 |
| 入口状态 | 角色带着什么问题/压力进入本幕 |
| 出口状态 | 角色带着什么不可逆变化离开本幕 |
| 关系阶段 | 核心关系处于：防备/误解/被迫靠近/试探/依赖/断裂/重建/承认 |
| 关系债务 | 本幕推进或加重哪一笔欠账，为什么不能在本幕完全兑现 |
| 亲密母题 | 本幕适合反复出现的抽象亲密行为（衣物/伤口/雨夜/共用物件/视线回避…） |
| 幕末钩子 | 事件层面的悬念 + 关系层面的余韵/缺口（两者都要有） |
| 预计章节数 | N章（附理由） |

## 四、节奏设计

跨幕次的节奏规划：
- 张弛曲线：哪些幕是高压、哪些幕是喘息、拐点在哪里
- 信息释放节奏：核心秘密/真相分几次揭露，每次揭多少
- 情绪色温变化：相邻幕次之间的温度差异（冷→热/暗→亮/压→释）

## 五、人物弧线规划

主角和核心角色跨幕次的变化轨迹：
- 每一幕角色的核心认知/状态发生什么变化
- 变化的触发事件是什么（不写具体剧情，写触发类型）
- 全书角色弧的方向：他从什么人变成什么人，代价是什么

## 六、伏笔与悬念设计

- 长线伏笔：在哪一幕埋、在哪一幕收、读者在中间幕次应该处于什么认知状态
- 短线悬念：每一幕内部的悬念如何接力（上一个解决时下一个已经冒头）
- 误导设计：哪些信息故意让读者形成错误预期，在哪里打破

## 七、时间线设计

- 故事内时间跨度：全书覆盖多长时间
- 每幕时间密度：本幕覆盖几天/几周/几月，节奏快慢
- 时间跳跃点：幕与幕之间是否有时间跳跃，跳跃后需要交代什么
