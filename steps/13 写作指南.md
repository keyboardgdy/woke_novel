设计生成一份写作指南，用于指导AI生成小说正文。

【设计原则】
1. 指南不重复剧情，剧情由剧情文件提供；指南只做执行层指令
2. 每条指令必须可执行——AI读完即知道该写什么、不该写什么，而非给方向但留解释空间
3. 风格用指纹不用标签（不说"古典雅致"，要给句长/词汇/修辞密度的具体刻度）
4. 情绪用纹理不用归类（不说"沉重压抑"，要写"克制的热/慢性的冷/闷在骨头里的焦灼"）
5. 禁忌要靶向不要泛泛（针对本故事的AI典型退化模式，不是通用写作常识）


【输出要求】
- 优先质量，不追求篇幅
- 本章正文目标控制在 2500-4500 字，默认按 3000 字左右设计；如果核心骨架或剧情文件给出更具体的章节字数安排，必须继承该安排
- 指南自身保持精炼，输出约 1500-2200 字，硬上限 2800 字；不要扩写成剧情复述或创作论文
- 指令要具体到可以照着写，不要留模糊空间
- 禁止项必须定制化，基于本故事的类型/人物/风格特征
- 严格使用下方章节结构

## 一、叙事坐标

1.1 定位锚点

- 当前位于第几幕第几场，引用剧情文件中的场景编号
- 前一场已落定的事件（一句话），下一场将接的入口（一句话）
- 本段在整体弧线中的功能位置（蓄力 / 转折 / 释放 / 回落）

1.2 场景锚点

- 具体地点（精确到房间 / 街道 / 时辰）
- 3-5个可作为描写支点的物件（标注功能：信息载体 / 情绪载体 / 氛围载体）
- 感官锚点（至少覆盖两个通道：视觉 / 听觉 / 嗅觉 / 触觉 / 味觉）

1.3 时间锚点

- 故事内时间（具体到时辰或事件发生后的第几天）
- 章节内时间跨度（这一段覆盖几分钟 / 几小时 / 几天）

## 二、风格指纹

2.1 句式特征

- 主用句长区间（短句为主 / 长短交替 / 长句为主）
- 句式偏好（是否使用文言句式 / 欧化句式 / 口语断句 / 倒装）
- 段落密度（一段一句 / 一段多层 / 场景内整段不分行）

2.2 词汇域

- 时代语域（古白话 / 近代白话 / 现代口语 / 混合）
- 职业 / 地域方言特征（是否带特定行话、方言词、术语）
- 禁用词方向（描述倾向，如"避免现代网络用语""避免古风过度堆砌"）

2.3 修辞密度

- 每千字的修辞事件数（比喻 / 通感 / 拟人等）
- 修辞类型偏好（意象型 / 反差型 / 感官型 / 哲思型）
- 是否有特殊修辞（反复 / 留白 / 断句 / 意识流片段）

2.4 情绪纹理

- 本段主导情绪的质感（不是"什么调"，是"什么质感"，如：克制的热、慢性的冷、闷在骨头里的焦灼）
- 主导情绪 + 1-2个变奏点（在哪个事件上情绪会发生转向）

## 三、人物声音

3.1 主要人物

- 说话方式（句尾习惯 / 用词偏好 / 语速节奏 / 是否经常欲言又止）
- 行为特征（标志性小动作 / 对特定物件的态度 / 身体语言模式）
- 本段内的心理状态（写"他在两个选择之间反复掂量，但开口时却选了第三个"，不写"他很纠结"）

3.2 关系动态

- 当前两人关系的实质（表层 / 里层是否一致）
- 本段关系是推进 / 凝固 / 倒退
- 关系变化的触发事件与触发后的状态
- 本段承接或新增的关系债务（亏欠 / 误伤 / 被看见 / 被选择 / 被抛下 / 被保护）
- 本段要凸显的男性/女性性感吸引力（力量感 / 柔软感 / 松弛感 / 危险感 / 克制感 / 成熟感 / 野性 / 脆弱 / 反差）
- 被吸引者会注意到的具体细节（姿态 / 声音 / 衣物 / 气味 / 动作节奏 / 体温 / 皮肤状态 / 肌肉反应 / 眼神停留）
- 本段的亲密边界（什么能做，什么不能做，为什么不能越过）
- 本段适合出现的抽象亲密行为（动作必须具体，服务人物关系）
- 本段是否允许性亲密真实发生；若允许，必须说明同意条件、叙述尺度和事后关系后果
- 亲密之后的防御反应（退开 / 玩笑 / 冷脸 / 转移话题 / 反客为主 / 沉默 / 索要确认 / 假装无事）

3.3 配角刻度

- 配角功能定位（信息传递 / 氛围烘托 / 对照 / 伏笔 / 工具人）
- 描写深度（有名有姓有几句台词 / 纯背景 / 一笔带过）
- 是否需要在本段建立配角的辨识度

## 四、情节执行

4.1 必达事件

- 引用剧情文件中的具体节点编号或场景名（不复述剧情内容）
- 每个节点的执行优先级：
    - 必须落地（核心事件，不可省略）
    - 可灵活处理（可调整顺序或合并）
    - 可裁剪（视篇幅可舍去）

4.2 信息密度控制

- 哪些信息要在本段首次透露（必须让读者明确接收到）
- 哪些信息要延后（读者可能猜到但不能写明）
- 哪些信息要模糊处理（留疑不解，给读者想象空间）

4.3 情绪弧线

- 起点状态（角色入场时的情绪）
- 转折触发点（哪个事件 / 哪句话 / 哪个细节会引发情绪转向）
- 终点状态（角色出场的情绪）
- 延迟释放点（情绪压在哪个位置故意不落，让它延后到下段）

4.3a 亲密张力执行

- 本段若包含男女旖旎感，必须通过距离、触碰前后、照顾、衣物、伤口、温度、气味、视线停留、共同沉默等抽象亲密行为呈现。
- 每个亲密瞬间必须包含三步：接近的理由、停住的边界、事后的反应。
- 亲密瞬间不能只让气氛变暧昧，还必须改变至少一项：主动权、信任、防备、亏欠、误解、依赖。
- 不要直接写"暧昧""心动""欲望""悸动""情愫暗生"来替代读者应当感受到的内容。

4.3a-1 性感吸引力描写

- 允许并鼓励描写成年男性或成年女性的性感吸引力，但必须通过人物视角中的注意力偏移呈现：他/她看见了什么、听见了什么、闻到了什么、为什么立刻移开视线或反而停住。
- 性感可以来自身体线条、力量感、柔软感、衣物贴合、声音压低、呼吸变化、湿发、伤口、汗意、气味、动作的控制感、松弛姿态、危险感、克制感或反差。
- 描写被吸引者的身体反应：话变少、喉咙发紧、动作慢半拍、视线避开又回来、突然刻薄、过度整理物件、靠近后才意识到距离。
- 同一段不要平均扫描全身；选择1-3个最有叙事功能的魅力细节，让它们改变对话、动作或关系主动权。
- 不要只写"美、帅、性感、诱人、火辣"；这些词可以偶尔出现，但不能替代具体观察。
- 不要把人物拆成身体部位清单。性感描写必须保留人格、处境、主动性和被观看者的边界。

4.3b 性亲密执行

- 如果剧情文件判断本段允许性亲密真实发生，正文不得用机械跳切回避事实；可以明确写出两人发生了性亲密。
- 性亲密必须具备：双方成年、清醒、自愿；同意可以含蓄，但必须可被读者辨认，且任何犹豫、暂停、拒绝都要被尊重。
- 描写重点放在选择、靠近、停顿、呼吸、身体余波、羞耻、防御、确认、事后沉默和关系变化，不写成器官、姿势、步骤或技术清单。
- 性亲密之后必须改变至少一项：信任、依赖、权力关系、亏欠、误解、自我认知、后续风险。
- 可以写不详细的发生过程，但不能让文本假装什么都没发生；事后必须留下可追踪的状态。

4.4 伏笔与回收

- 引用剧情文件中已标注的伏笔编号（哪些要在本段回收）
- 本段新埋的伏笔（具体载体：物件 / 对话 / 细节动作，不写大纲只写载体）
- 伏笔的伪装程度（一眼可见 / 细想可察 / 完全隐蔽）

4.5 落幕方式

- 本段结尾的悬念形态（未说完的话 / 未揭开的真相 / 未到达的人 / 情绪反刍 / 场景骤停）
- 末句的情绪走向（上扬 / 下沉 / 平收 / 反转）
- 末句的句式特征（短促 / 反问 / 未完成 / 意象收束）

## 五、节奏控制

5.1 篇幅与结构

- 正文目标字数区间：明确写出上下限，默认 2500-4500 字；不要写“适中”“较长”等模糊词
- 内部节拍数控制在 3-5 个，每个节拍写出大致字数占比
- 每个节拍必须标注“必写 / 可压缩 / 可删除”，方便正文生成时在超出字数时优先裁剪
- 如果剧情信息量过大，优先压缩说明、铺垫、重复心理和非关键环境描写，不得压缩核心转折、关系变化和章节落点

5.2 段落类型与节奏分配

- 动作段 / 心理段 / 对话段 / 留白段 在本段中的位置与配比
- 慢镜头 / 快进 / 留白 的具体落点
- 张弛比例（如：3拍紧 + 1拍松 / 整体张 / 整体弛）

## 六、靶向禁忌

6.1 禁忌清单

- 针对本故事定制，每条一句话，具体到可识别
- 不留通用例子，不重复AI默认的写作常识
- 形式：禁止 + 原因（为什么这条对本故事特别重要）

6.2 高风险位置

- 本段哪里最容易写崩（节奏 / 情感 / 信息 / 风格）
- 每个风险点给一句预警（写到这里要注意什么）

## 七、表达控制

7.1 呈现优先级

情绪表达顺序：

动作
＞细节
＞对话
＞心理
＞结论

优先展示现象，不直接解释情绪。

7.2 人物真实度

允许：

- 犹豫
- 误判
- 话说一半
- 答非所问
- 临时改主意

禁止全知全能式思考。

7.3 细节规则

每个核心场景至少包含：

- 1个人物细节
- 1个环境细节
- 1个物件细节

细节必须参与叙事。

禁止纯装饰描写。

7.4 AI退化规避

禁止：

- 情绪直接定义
- 剧情汇报式叙述
- 角色突然转变立场
- 为爽点牺牲逻辑
- 为推进剧情降低角色智商
- 用香艳标签、直白心理总结或机械性描写冒充旖旎感
- 用"很美/很帅/很性感"等空泛标签替代男性或女性的具体吸引力
- 把性感描写写成全身扫描、身体部位清单或与人物处境无关的凝视
- 让亲密行为无代价、无边界、无后续反应
- 性亲密发生后无事发生、无人变化、关系没有后果

所有变化必须存在可见触发链。
