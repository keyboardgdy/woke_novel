Session 1

Run Step 01 Creative Proposal Generation in a loop for `option_count`, producing `option_count` creative proposals. The user selects one proposal. Then run Step 02 Creative Proposal Supplement and append the result to `Creative_Proposal_{option_index}.md`.

After selection, extract the novel title and rename the project directory to the final novel title. Extract reference works and write them into the project profile.

Session 2

Run Step 03 Worldbuilding and Setting Generation. Run Step 04 Character Profiles and Relationship Matrix.

Session 3

Run Step 05 Story Axis. Run Step 05a Act Framework. Run Step 05b Act Core Skeleton once for each act.

After all Step 05b runs are complete, run Step 18 with `phase=post_05b` to generate the project-root `CLAUDE.md`.

Session 4

Run Step 06 Opening Plot Direction Extraction, outputting `Plot_Direction_v1.md`. Run Step 07 Opening Chapter Synopsis, outputting `Plot_v1.md`. Run Step 08 Opening Writing Guide, outputting `Writing_Guide_v1.md`. Run Step 09 Opening Chapter Draft, outputting `Draft_v1.md`. Run Step 10 State Document, outputting `State_v1.md`.

After the opening chapter is complete, use `extract_and_create_story_summary` to write the first chapter synopsis into the initial version of `{state}/Story_Summary.md`.

Creation Loop (rounds start from 2)

Iterate by act and chapter count. Each act has a specified number of chapters. Since the opening chapter of Act 1 is already complete, Act 1 loops `act_chapters - 1` times. Other acts loop `act_chapters` times.

Each round runs: Step 11 Plot Direction Guidance, outputting `Plot_Direction_v{round}.md`; Step 12 Plot Synopsis Design, outputting `Plot_v{round}.md`; Step 13 Writing Guide, outputting `Writing_Guide_v{round}.md`; Step 14 Chapter Draft, outputting `Draft_v{round}.md`; Step 15 State Document, outputting `State_v{round}.md`; Step 16 Story Summary Compression, run immediately after 15 inside the round session, outputting to `{state}/Story_Summary.md`. Then `sync_summary_to_state` writes the compressed version back into the final state document for the round, `State_v{round}.md`.

After an act is complete, run Step 17 Act-Level Story Summary Compression, outputting to `{state}/Story_Summary.md`, and use `sync_summary_to_state` to write it back into the final state document of the act. Then run Step 18 with `phase=post_17` to refresh the project-root `CLAUDE.md`.

## Directory Structure

```text
{project}/
|-- 00_baseline/     # Creative proposals and foundation settings
|-- 01_plots/        # Plot-related files, including opening Plot_Direction_v1.md
|-- 02_guides/       # Writing guides
|-- 02_output/       # Draft output
|-- 03_state/        # State documents and story summary
|-- 04_characters/   # Character profiles
`-- CLAUDE.md        # Project-level instructions generated by Step 18 (a file, not a directory)
```

> Logs are located at the repository root under `logs/<novel title>/`, parallel to `projects/`, not inside the project directory.
